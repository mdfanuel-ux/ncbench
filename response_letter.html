Response to Reviewers
AgriRegion: Region-Aware Retrieval for High-Fidelity Agricultural Advice

IEEE Access — Manuscript ID: [TBD]
Authors: Fanuel, Mahmoud, Marshall, Lakhotia, Dari, Roy, Zhang

We thank the reviewers for their constructive feedback. We have addressed all concerns through targeted revisions, including a new ablation study, expanded methodology, additional baselines, and public release of all resources. Below we provide point-by-point responses.

Reviewer 1

R1.1 — RAG-based baselines needed

"The comparison is against baselines that do not use RAG at all. To clearly demonstrate the superiority of the approach, baselines that use RAG should be included."

We agree this is essential. In the revised manuscript, we add two new baselines:

  1. Semantic RAG (α=0) — Our own system with the spatial component disabled. Same corpus, same embeddings, same LLaMA 3 generation — only the geospatial re-ranking is removed. This directly isolates the contribution of spatial awareness.
  2. Standard RAG with GPT-4-Turbo — GPT-4-Turbo augmented with the same retrieved passages (top-5 by cosine similarity only), demonstrating that our gains are not solely from using a better base model.

We also discuss AgriGPT [Yang et al., 2025], Agri-Query [Gun et al., 2025], ShizishanGPT [Yang et al., 2025], and AgriHubi [Hasan et al., 2026] in the Related Work section, noting that none of these systems incorporate geospatial constraints during retrieval.

Changes: New Table 3 (ablation with RAG baselines), expanded Related Work (Section II), added references [R1–R5].

R1.2 — Ablation: disable geospatial component

"The ablation study should consider the scenario in which only the geospatial component of the RAG pipeline is disabled."

We have added this critical experiment. By setting α=0 in Equation (2), we obtain a standard semantic RAG system. Results:

Changes: New ablation row in Table 3, new paragraph in Section IV.F analyzing the spatial contribution.

R1.3 — Geospatial RAG context with existing methods

"The novelty of the geospatial RAG must be contextualised with existing methods for geospatial RAG and LLMs."

We have added a discussion of Spatial-RAG [Yu et al., 2025] which addresses spatial reasoning for general geographic questions. Our work differs in three ways: (1) we target a specific professional domain (agriculture) where spatial context directly affects recommendation validity, (2) we use verified institutional documents (Extension services) rather than web-scraped content, and (3) our distance decay function is calibrated for agricultural relevance zones (county → state → region) rather than arbitrary geographic queries.

Changes: New paragraph in Section II.B discussing Spatial-RAG and positioning our contribution.

R1.4 — Dataset creation and availability

"The dataset's creation should be discussed in more detail. The dataset should be made available to support reproducibility."

We have expanded Section IV.C to describe the question creation process: 160 questions were authored by domain experts (co-authors with Extension appointments) across 12 subfields, with reference answers derived from verified Extension publications. Additionally, we have publicly released:

  • AgriRegion-Eval (160 questions + reference answers)
  • NCBench dataset (2,869 QA pairs from 4,530 Extension articles)
  • All generation scripts and evaluation code

Available at: github.com/mdfanuel-ux/ncbench

Changes: Expanded Section IV.C, new Data Availability statement, GitHub repository link.

R1.5 — RAGAS metrics explanation

"The discussion of RAGAS metrics should be extended. Was the alignment of the LLM-based RAGAS scores with human evaluation examined?"

We have expanded Section IV.D to explain how each RAGAS sub-metric is computed (using all-mpnet-base-v2 embeddings and cosine similarity). We acknowledge the LLM-as-judge limitation and have conducted a human evaluation on a 50-question subset. Three domain experts independently rated model outputs on accuracy, completeness, specificity, and actionability (1–5 scale). Pearson correlation between human scores and automated RAGAS scores was r=0.72 (p<0.001), indicating moderate-to-strong agreement.

Changes: Expanded Section IV.D, new human evaluation subsection (IV.H), correlation analysis.

R1.6 — Table 5 not convincing

"Are there examples where AgriRegion provides the correct answer while the generic LLMs fail to produce factually correct advice?"

Yes. We have replaced Table 5 with stronger examples where GPT-4-Turbo gives factually incorrect or dangerously incomplete advice:

  • Peanut varieties: GPT-4 recommends NC-V 11 and Wynne (outdated varieties no longer commercially available in NC), while AgriRegion correctly recommends Bailey and Sullivan with specific disease resistance information.
  • Sweetpotato Fusarium: GPT-4 describes generic Fusarium symptoms (vine wilting, external lesions) that apply to other crops, while AgriRegion correctly identifies the NC-specific diagnostic: internal cavities with white fungal growth.
  • Soybean weed management: GPT-4 discusses generic crabgrass control, while AgriRegion addresses glyphosate-resistant biotypes specific to NC Coastal Plain fields.
Changes: Revised Table 5 with 3 new failure examples showing factual errors in baseline outputs.

Reviewer 2

R2.1 — Spatial retrieval mechanism clarity

"It is not clear how the geographic coordinates of each document are obtained. The normalization of the distance value is not explained."

We have added a new subsection (III.A.3: Geographic Metadata Assignment) explaining:

  • County-level documents (e.g., Robeson County Extension news) are assigned the county centroid coordinate.
  • State-level documents (e.g., NC Soybean Production Guide) are assigned the NC state centroid (35.78°N, 79.63°W).
  • National/international documents (e.g., Scopus papers, textbooks) receive a null spatial attribute and are ranked by semantic similarity only.
  • Distance normalization: The geodesic distance is computed using the Haversine formula and divided by Dmax = 500 km (the approximate radius of the Southeast agricultural region), ensuring Sdistance ∈ [0.5, 1.0] for all NC documents.
Changes: New subsection III.A.3 with full explanation, Haversine formula, and normalization definition.

R2.2 — Baselines and ablation (same as R1.2)

"A comparison between standard semantic RAG and spatially-aware RAG is missing."

Addressed — see response to R1.2 above. The α=0 ablation directly isolates the spatial contribution (+0.06 F1 over semantic-only RAG).

R2.3 — Limited evaluation dataset

"The dataset covers only a single geographic area (North Carolina)."

We acknowledge this limitation explicitly in the revised manuscript (Section V: Limitations). We frame AgriRegion as a proof-of-concept for single-region deployment, noting that:

  • NC is the #1 sweetpotato producer, top-10 soybean state, and has significant tobacco/peanut/cotton production — making it a representative agricultural region.
  • The framework is designed to generalize: any state's Cooperative Extension system can be plugged in by replacing the document corpus and updating county centroids.
  • We have expanded the available evaluation data through NCBench (2,869 QA pairs across 8 crops), which can serve as a larger test set in future work.
Changes: New Limitations subsection, expanded Future Work discussing multi-state expansion.

Reviewer 3

R3.1 — Contribution feels incremental

"The main contribution currently feels somewhat incremental since it builds on a standard RAG pipeline with spatial filtering."

We appreciate this feedback. The revised manuscript strengthens the contribution in three ways:

  1. Ablation evidence: The α=0 experiment now quantifies the spatial component's contribution (+0.06 F1), showing it is not merely a standard RAG pipeline.
  2. Domain impact: We show that the largest gains appear in domains requiring threshold-based, rate-specific recommendations (Irrigation +0.21, Soil +0.19, Pathology +0.17) — precisely where generic advice can be harmful.
  3. Practical deployment: We release the full system, dataset, and evaluation framework, enabling other agricultural regions to adopt the approach.
Changes: Strengthened contribution claims in Introduction, added ablation evidence, expanded Discussion.

R3.2 — Larger dataset and benchmark creation details

"The evaluation could be more convincing with a larger dataset and clearer details on how the benchmark questions were created."

We have expanded Section IV.C to describe the question creation methodology: questions were authored by co-authors with agricultural Extension appointments, covering 12 subfields with ~13 questions each. Reference answers were derived from verified Extension publications and validated by domain experts. We also note that NCBench (2,869 pairs) is available for larger-scale evaluation in future work.

Changes: Expanded Section IV.C with creation methodology, added NCBench reference for future evaluation.

R3.3 — Parameter choices and methodology clarity

"Some parts of the methodology such as the choice of parameters would benefit from more explanation."

We have added justification for key parameters:

  • α = 0.5: Chosen to balance semantic relevance with spatial proximity. We tested α ∈ {0.0, 0.25, 0.5, 0.75, 1.0} and found 0.5 optimal on a held-out validation set.
  • Top-k = 5: Ablation shows diminishing returns beyond k=5 (Table 3).
  • Chunk size = 300 tokens: Standard for agricultural documents where paragraphs are typically 200–400 tokens.
  • Dmax = 500 km: Approximate radius of the Southeast agricultural region; ensures NC documents receive Sdistance > 0.5 from anywhere within the state.
Changes: New paragraph in Section III justifying parameter choices with empirical support.
