🌍 AgriRegion

Region-Aware Retrieval for High-Fidelity Agricultural Advice
Mesafint Fanuel, Mahmoud Nabil Mahmoud, Crystal Cook Marshall, Vishal Lakhotia, Biswanath Dari, Kaushik Roy, Shaohu Zhang
IEEE Access (Under Revision) · NC A&T State University · University of Alabama

Abstract

LLMs frequently suffer from "contextual hallucination" in agriculture — providing advice that is scientifically sound in one region but disastrous in another due to variations in soil, climate, and local regulations. AgriRegion is a RAG framework that incorporates a geospatial metadata injection layer and a region-prioritized re-ranking mechanism. By restricting the knowledge base to verified local Extension documents and enforcing geospatial constraints during retrieval, AgriRegion ensures locally accurate advice.

Key Result: AgriRegion reduces hallucinations by 10–20% compared to GPT-4-Turbo, Claude 3.5 Sonnet, Gemini 1.5 Pro, and Mistral-7B on 160 agricultural questions across 12 subfields.

Spatial-Semantic Retrieval (Key Innovation)

Standard RAG retrieves by semantic similarity alone. AgriRegion adds a distance decay function:

Sfinal = (1 − α) · Ssemantic + α · Sdistance

Sdistance = 1 / (1 + d(guser, gdoc))

Where d is normalized geodesic distance. NC Extension docs get Sdistance ≈ 1 for NC users. Out-of-region docs are penalized but not eliminated.

🔴 CRITICAL — Ablation: Semantic RAG vs Spatial RAG

Reviewer concern: Current comparison is AgriRegion (RAG + spatial) vs bare LLMs (no RAG). This is unfair — gains might come from RAG itself, not the spatial component.

The question: Does spatial awareness improve RAG, or is any RAG system equally good?

Experiment Design:

  1. Set α = 0 → standard semantic RAG (no spatial component)
  2. Keep everything else identical: Ada v2 embeddings, same corpus, same pipeline, same eval
  3. Compare: Semantic RAG (α=0) vs Spatial RAG (α=0.5) vs No RAG

Expected outcome: Semantic RAG improves over No RAG by ~0.10 F1. Spatial RAG improves over Semantic RAG by additional 0.05–0.10 F1. This proves the geo component adds value beyond just having RAG.

🟠 HIGH — Spatial Retrieval Mechanism (Clarity Fix)

Reviewer concern: How are coordinates assigned? How are region-based documents handled? What's the normalization?

This is a presentation gap, not a conceptual flaw.

Revised Explanation (for paper):

  • State-level docs (NC Extension publications) → assigned NC centroid (35.78°N, 79.63°W)
  • County-level docs (county Extension news) → assigned county centroid
  • National/international docs (Scopus papers, textbooks) → null spatial attribute, ranked by semantic only
  • Distance computation: Haversine formula between user location and document coordinate
  • Normalization: Divided by maximum regional threshold (e.g., 500 km for Southeast US) to keep values in [0, 1]
